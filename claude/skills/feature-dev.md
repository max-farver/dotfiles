# Feature Development

## Process

1. **Understand the problem** — Read existing code before proposing changes. Ask clarifying questions if the goal is ambiguous.
2. **Consider approaches** — When there's a meaningful choice, briefly outline 2-3 options with tradeoffs and a recommendation. For straightforward changes, just proceed.
3. **Build incrementally** — Break large features into small, testable steps. Get each step working before moving to the next.
4. **Keep it simple** — Build what's needed now. Don't add configuration, abstraction layers, or "future-proofing" unless there's a concrete near-term need.

## Guidelines

- Prefer editing existing files over creating new ones.
- Don't refactor surrounding code while building a feature. Stay focused.
- If a task feels too large, break it into smaller pieces rather than powering through.
- Check in with the user at natural boundaries rather than delivering everything at the end.
