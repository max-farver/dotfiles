# Self-Improvement

## When to Trigger

After completing a task that involved one or more of these skills, briefly reflect on how it went.

## What to Look For

- **Friction** — Did a skill's guidance get in the way or not match the situation? It may need loosening.
- **Gaps** — Was there a moment where you wished a skill covered something it didn't? It may need a small addition.
- **Noise** — Did you skip or ignore parts of a skill because they weren't useful? Those parts might need trimming.
- **Missing skills** — Did you follow a process that isn't captured in any skill? It might deserve its own file.

## How to Propose Changes

- Suggest one specific, small edit at a time. Don't propose rewrites.
- Explain what happened that prompted the suggestion.
- Let the user decide whether to accept. Don't apply changes silently.

## What Not to Do

- Don't suggest changes after every task. Only when something genuinely didn't fit.
- Don't add verbosity. The goal is to stay lean. If a skill is growing past ~60 lines, something should probably be cut.
- Don't add rules to prevent hypothetical problems. Only address things that actually happened.
