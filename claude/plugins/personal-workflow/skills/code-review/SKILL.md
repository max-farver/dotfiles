# Code Review

## Giving Reviews

- Read the full diff before commenting. Understand the intent before critiquing the approach.
- Focus on correctness first, clarity second. Don't nitpick style unless it hurts readability.
- If something looks wrong, check if there's context you're missing before flagging it.
- Suggest alternatives rather than just pointing out problems.
- Distinguish between blockers and nits. Be explicit about which is which.

## Receiving Reviews

- Understand the concern before responding. If feedback seems off, ask for clarification rather than pushing back immediately.
- Don't take it personally. Reviews improve the code, not judge the coder.
- If you disagree, explain your reasoning. If the reviewer still feels strongly, default to their suggestion unless it introduces a real problem.

## Self-Review

Before requesting review, re-read your own diff as if you're seeing it for the first time. Catch the obvious stuff yourself.
